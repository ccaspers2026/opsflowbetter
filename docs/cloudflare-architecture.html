Image Upload Flow (orange)

1. You drop an image in the Image Fixer modal
2. Browser processes it (bg removal + white composite)
3. Pipeline sends PUT /upload/45-rpm-adaptor/main.jpg to Worker
4. Worker authenticates via X-Upload-Secret header
5. Worker stores JPEG in R2 bucket — overwrites if exists
6. Worker returns public URL → Pipeline auto-saves it

Image Serving Flow (green)

1. Pipeline or Portfolio page loads <img> tag
2. Browser requests images.opsflowbetter.com/45-rpm-adaptor/main.jpg
3. Cloudflare R2 serves the image via its custom domain
4. Cached at Cloudflare's edge — fast worldwide delivery

Code Deploy Flow (purple)

1. You commit in VS Code and git push
2. GitHub receives code → auto-deploys to GitHub Pages
3. Cloudflare DNS proxies opsflowbetter.com → GitHub Pages
4. Users see the updated site with SSL via Cloudflare
