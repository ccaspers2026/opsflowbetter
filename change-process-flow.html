⟳ Rework Loop
Same version — no version bump. Claude fixes code, provides updated commit.
Christian deploys. Claude re-tests ONLY failed items.
Repeat until all PASS or SKIP. HALT banner stays RED throughout.
↑ Back to Phase 4 (Commit Handoff)

Non-Negotiable Rules
No code before GO. Discussion and planning are free. Executing is expensive.
No commit message before COMMIT. Christian controls when changes ship.
Claude never commits or pushes. Commit message in code block only. Christian uses Visual Studio.
Rework stays in version. No version bump during rework. Only re-test failed items.
Bookend scans every cycle. Check for new feedback/tasks before AND after work.
All 9 pages get version bump. Every code change (unless rework) bumps all pages.
HTML + localStorage in sync. Always update both — HTML is the safety net fallback.
When in doubt, read the doc. Don't guess from faded context.
